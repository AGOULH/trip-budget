# خطوات البناء في التيرمنل (Claude Code) — حاسبة ميزانية الرحلات

> ضع `CLAUDE.md` و هذا الملف في جذر المشروع. Claude Code يشتغل داخل التيرمنل.
> repo: `trip-budget` · النشر: `agoulh.github.io/trip-budget/`

## ١) التهيئة (Terminal)

```bash
# تأكد من تثبيت Claude Code (مرة واحدة)
npm install -g @anthropic-ai/claude-code

# إنشاء المشروع
mkdir -p ~/projects/trip-budget && cd ~/projects/trip-budget
# انسخ CLAUDE.md و خطوات_البناء.md إلى هنا
git init
gh repo create trip-budget --public --source=. --remote=origin   # أو أنشئه يدويًا
```

Firebase: أنشئ مشروعًا + فعّل Firestore، وجهّز الـweb config (ستلصقه في المرحلة 0). قواعد مبدئية:
```
rules_version='2';
service cloud.firestore { match /databases/{db}/documents {
  match /{document=**} { allow read, write: if true; } } }   // شدّدها لاحقًا
```

## ٢) الطريقة الموصى بها — جلسة تفاعلية

```bash
cd ~/projects/trip-budget
claude -n "trip-budget"
```

تفتح جلسة داخل التيرمنل. الصق أوامر المراحل التالية **واحدًا تلو الآخر**، وراجع اختبار القبول قبل ما تنتقل. لو خرجت وتبي تكمل لاحقًا:

```bash
claude -c          # يكمل آخر جلسة في هذا المجلد
# أو
claude -r "trip-budget"
```

### أوامر المراحل (تُلصق داخل الجلسة)

**(0) الإقلاع**
```
اقرأ CLAUDE.md بالكامل. نفّذ المشروع كصفحة واحدة index.html عربية RTL (خط Cairo) + Firebase Firestore (SDK modular عبر CDN). أنشئ الهيكل: تهيئة Firebase بهذا الـconfig [ألصق config]، تنسيق RTL عام، وتنقّل بسيط بين الشاشات (SPA). لا تبنِ منطق الحساب بعد. القبول: الصفحة تفتح وتتصل بـFirestore بلا أخطاء.
```
**(1) الرحلات** — `نفّذ المرحلة 1 من CLAUDE.md: قائمة الرحلات + إنشاء رحلة (اسم/سنة/عملات/محصّل/أسعار الصرف) بالحفظ في trips. القبول: أنشئ رحلة وتظهر في القائمة.`

**(2) المجموعات والمشاركين** — `نفّذ المرحلة 2: مجموعات محفوظة (groups) + بناء قائمة مشاركي الرحلة (مجموعة كاملة/اختيار/يدوي) + حذف سهل + "حفظ كمجموعة جديدة". قائمة الرحلة نسخة مستقلة. القبول: أعدّل قائمة الرحلة دون تأثّر المجموعة الأصلية.`

**(3) أسعار الصرف** — `نفّذ المرحلة 3: إدخال يدوي (USD, IQD مقابل الريال) + زر "جلب لحظي" من https://open.er-api.com/v6/latest/SAR قابل للتعديل، وتُخزَّن داخل وثيقة الرحلة. القبول: تُحفظ الأسعار.`

**(4) المصاريف** — `نفّذ المرحلة 4: إضافة/إدارة المصاريف (subcollection expenses) بحقول CLAUDE.md، تحديد المشاركين "النشطين فقط" + تحديد الكل + فرز/فلترة + تعديل/حذف. القبول: أسجّل مصروفًا بعملة على مجموعة فرعية.`

**(5) محرّك الحساب** — `نفّذ المرحلة 5: تحويل كل مصروف للريال، نصيب الفرد بالتساوي، ورصيد كل شخص (دفع − نصيب)، والتحقق أن مجموع الأرصدة = صفر. القبول: الأرصدة صحيحة ومتوازنة.`

**(6) الإقفال الجزئي** — `نفّذ المرحلة 6: "إقفال الميزانية" (اختيار مجموعة فرعية → معاينة → تأكيد → تجميد في closures مع snapshot + settlementLines، وتعليمهم closed+closedStage) + "إعادة فتح" + زر "تم التحصيل". بعد الإقفال المصاريف الجديدة تعرض النشطين فقط. القبول: أقفل مجموعة فيتجمّد رصيدها وتُستبعد لاحقًا، وأعيد فتحها.`

**(7) التسوية** — `نفّذ المرحلة 7: تسوية عبر المحصّل (مدين→محصّل، محصّل→دائن) + عرض بديل "أقل تحويلات". القبول: الجدول صحيح في الوضعين.`

**(8) الداش بورد** — `نفّذ المرحلة 8: داش بورد Chart.js (بطاقات + صرف حسب التصنيف/العملة + دفع مقابل نصيب + منحنى الأيام + حالة الإقفال). القبول: الرسوم حيّة.`

**(9) التقارير** — `نفّذ المرحلة 9: تقرير كل شخص (دفع/نصيب/الصافي/تفصيل بالتصنيف والعملة والمُحوَّل للريال + سطر التسوية + وسم الإقفال) بصيغتَي Word (مكتبة docx) و PDF (html2pdf.js بخط عربي مضمّن) + "تصدير الكل" ZIP. القبول: تصدير شخص بصيغتين بعربي سليم.`

**(10) اللمسات + النشر** — `نفّذ المرحلة 10: تحسينات + PWA اختياري + تجهيز النشر على GitHub Pages من index.html في الجذر.`
