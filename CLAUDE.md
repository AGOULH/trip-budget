# حاسبة ميزانية الرحلات الجماعية — ملف الأساس (CLAUDE.md)

> ملف مواصفات المشروع. يُوضع في جذر مجلد المشروع ويُبنى عليه في Claude Code.
> نمط العمل: صفحة واحدة عربية RTL (HTML/CSS/JS) + Firebase Firestore، نشر على GitHub Pages (`agoulh.github.io`).

---

## 1) نظرة عامة

تطبيق ويب لإدارة مصاريف رحلة جماعية (زيارات/سفر)، حيث:

- **كل مصروف يُقسَّم فقط على من شارك فيه** (وليس على كل المجموعة).
- التقسيم **بالتساوي** بين المشاركين في المصروف الواحد.
- **الرحلة تتدرّج بمراحل**، وبعض المشاركين قد ينسحبون في منتصفها → **إقفال ميزانية جزئي**.
- يدعم **٣ عملات:** ريال سعودي (SAR)، دولار (USD)، دينار عراقي (IQD).
- **التقرير النهائي يحوّل كل شيء إلى الريال السعودي** (العملة المرجعية).
- إخراج **تقرير لكل شخص** (Word + PDF) + **داش بورد** للمصاريف.
- **تسوية عبر محصّل/أمين صندوق:** من عليه مبلغ يحوّل للمحصّل، والمحصّل يوزّع لمن له.

اللغة: عربي، الاتجاه RTL. أسماء الحقول والدوال في الكود بالإنجليزي.

## 2) المفاهيم الأساسية (مهم جدًا)

### أ) المجموعة المحفوظة (Saved Group / Roster)
قائمة أشخاص قابلة لإعادة الاستخدام (مثال: "شباب النجف" = ١٠ أشخاص). تُحفظ مرة وتُستعمل في أكثر من رحلة.

### ب) الرحلة (Trip)
لها **قائمة مشاركين خاصة بها فقط**. عند الإنشاء تُبنى القائمة إما:
- **بإضافة مجموعة محفوظة كاملة**، أو **باختيار أشخاص** منها، أو **بإضافة أشخاص جدد** يدويًا.

قاعدة حاسمة: تعديل قائمة الرحلة **لا يؤثر على المجموعة المحفوظة** (نسخة مستقلة "لهذه السفرة فقط"). عند اختلاف التركيبة يظهر زر **"حفظ كمجموعة جديدة"**.

### ج) التدرّج والإقفال الجزئي (Stages & Partial Closing) — الجديد
الرحلة سلسلة مراحل بالترتيب (مثال: **تذاكر → فيزا → سكن → مواصلات داخلية → …**). ليس كل المشاركين يكملون كل المراحل:

- عند نقطة معيّنة، مجموعة فرعية تنسحب → **نقفل ميزانيتهم**: يُحسب حسابهم النهائي حتى تلك اللحظة، يُولَّد تقريرهم، ويُعلَّمون **"مقفل"**.
- بقية المجموعة (الأشخاص **النشطون**) يكملون للمرحلة التالية.
- المصاريف السابقة المشتركة (تذاكر/فيزا) تبقى مقسّمة على من شارك فيها فعلًا وقتها؛ الإقفال فقط **يجمّد** رصيد المنسحب و**يستبعده** من المصاريف اللاحقة.

## 3) تدفّق العمل (Flow)

1. إنشاء رحلة → **إضافة الأشخاص أولًا** (مجموعة كاملة / اختيار / يدوي).
2. تحديد المحصّل وإدخال أسعار الصرف.
3. **بعدها تسجيل المصاريف** بالتدرّج: تذاكر، فيزا، سكن…
4. عند انسحاب مجموعة فرعية → **إقفال ميزانيتهم** وتوليد تقاريرهم.
5. الاستمرار مع النشطين حتى نهاية الرحلة → إقفال نهائي للجميع + تقارير + تسوية.

## 4) نموذج البيانات (Firestore)

```
groups (collection)              ← المجموعات المحفوظة القابلة لإعادة الاستخدام
  {groupId}
    name: string
    members: [{ id, name, phone? }]
    createdAt, updatedAt

trips (collection)               ← الرحلات
  {tripId}
    name: string                 // مثال: "زيارة كربلاء 2027"
    year: number
    baseCurrency: "SAR"
    currencies: ["SAR","USD","IQD"]
    rates: { USD: number, IQD: number }   // كم يساوي 1 من العملة بالريال
    ratesSource: "manual" | "api"
    ratesUpdatedAt: timestamp
    treasurerId: string
    participants: [{
      id, name, phone?,
      status: "active" | "closed",   // نشط / مقفل
      closedAt: timestamp | null,
      closedStage: string | null     // مثال: "بعد الفيزا"
    }]
    sourceGroupId: string | null
    createdAt, updatedAt

    expenses (subcollection)     ← مصاريف الرحلة
      {expenseId}
        desc: string
        category: string         // تذاكر / فيزا / سكن / مواصلات / أكل / زيارات / متفرقات
        amount: number           // بالعملة الأصلية
        currency: "SAR"|"USD"|"IQD"
        date: string (ISO)
        payerId: string          // من دفع
        participantIds: [string] // من يشملهم هذا المصروف تحديدًا (نشطون وقتها)
        splitMethod: "equal"
        createdAt

    closures (subcollection)     ← سجلّات إقفال الميزانية (الجديد)
      {closureId}
        participantIds: [string]     // من أُقفلت ميزانيتهم في هذه الدفعة
        stageLabel: string           // مثال: "بعد الفيزا"
        closedAt: timestamp
        snapshot: [{ participantId, paidSAR, shareSAR, netSAR }]  // أرصدة مجمّدة
        settlementLines: [{ fromId, toId, amountSAR }]
        collected: boolean           // هل تم التحصيل فعليًا؟
        reportGenerated: boolean
```

`participants` مضمّنة داخل وثيقة الرحلة (snapshot) لضمان استقلالها عن المجموعة المحفوظة.

## 5) محرّك الحساب والتسوية

### التحويل للريال
لكل مصروف: `amountSAR = amount × rate[currency]` (وسعر الريال = 1).

### نصيب الفرد
```
sharePerPerson = amountSAR ÷ participantIds.length   // بالتساوي
```

### رصيد كل شخص
```
balance(person) = (مجموع ما دفعه بالريال) − (مجموع أنصبته بالريال)
```
موجب → **له** (دائن) | سالب → **عليه** (مدين) | مجموع الأرصدة = صفر (تحقّق).

### قواعد الإقفال الجزئي
- عند **إضافة مصروف** يظهر في قائمة المشاركين **النشطون فقط** افتراضيًا؛ المقفلون مخفيّون (أو مع تحذير لو حاولت إضافتهم).
- رصيد المقفل **مجمّد** من لحظة الإقفال (يُخزَّن في `snapshot`).
- لا يُحتسب المقفل في أي مصروف تاريخه بعد إقفاله (تحذير إن حصل).

### التسوية — الوضع الافتراضي (محصّل)
- كل مدين → يحوّل قيمته للمحصّل. المحصّل → يوزّع على كل دائن.
- يُعرض جدول: "فلان → المحصّل: X" ثم "المحصّل → فلان: Y".

### التسوية — وضع بديل (أقل عدد تحويلات)
خوارزمية جشعة: طابِق أكبر مدين مع أكبر دائن تكرارًا. عرض اختياري.

## 6) الشاشات (UI)

1. **قائمة الرحلات:** بطاقات (الاسم، السنة، المشاركون النشطون/الكل، إجمالي الصرف بالريال) + "رحلة جديدة".
2. **إنشاء رحلة:** الاسم/السنة → مصدر المشاركين → المحصّل → أسعار الصرف.
3. **إدارة المشاركين:** إضافة/حذف سهل، شارة حالة (نشط/مقفل)، زر "حفظ كمجموعة جديدة"، زر **"إقفال ميزانية"** و**"إعادة فتح"**.
4. **إضافة مصروف:** الوصف، التصنيف، المبلغ + العملة، التاريخ، الدافع، تحديد المشاركين (النشطون؛ مع "تحديد الكل").
5. **قائمة المصاريف:** جدول قابل للفرز/الفلترة (تصنيف/عملة/شخص) + تعديل/حذف.
6. **إقفال الميزانية:** اختيار المجموعة الفرعية → **معاينة حسابهم النهائي والتسوية** → تأكيد → توليد التقارير + تعليمهم "مقفل".
7. **الداش بورد:** (القسم 9).
8. **التقارير:** تقرير كل شخص + تقرير التسوية.
9. **إعدادات أسعار الصرف:** يدوي + "جلب السعر اللحظي".

---

## 7) العملات وسعر الصرف

- **افتراضي: إدخال يدوي** لكل رحلة (يطابق سعر الصرّاف الفعلي — الأدق).
- **اختياري: جلب لحظي مجاني بدون مفتاح** من `https://open.er-api.com/v6/latest/SAR` (يغطي USD و IQD) كنقطة بداية قابلة للتعديل.
- تُخزَّن الأسعار داخل وثيقة الرحلة وقت الإنشاء فلا تتغيّر التقارير لاحقًا.
- `rate[currency]` = كم يساوي 1 وحدة من العملة **بالريال** (SAR = 1).

## 8) التقارير (Word + PDF)

تقرير كل شخص يشمل: إجمالي ما دفعه، إجمالي نصيبه، **الصافي (له/عليه)**، تفصيل حسب التصنيف/المرحلة، تفصيل بالعملة الأصلية + المُحوَّل للريال، قائمة مصاريفه، وسطر التسوية (تحوّل للمحصّل X / تستلم منه Y). ويُوسَم إن كان **مقفلًا** والمرحلة.

**التنفيذ:**
- **Word:** مكتبة `docx` (JS) بعربي RTL.
- **PDF:** من HTML عبر `html2pdf.js` مع خط عربي مضمّن (Amiri/Cairo). تجنّب jsPDF النصي.
- "تصدير الكل" = تقارير الأشخاص دفعة واحدة (ZIP)، ويشمل تقارير الدفعات المقفلة سابقًا.

---

## 9) الداش بورد (Chart.js)

- بطاقات: إجمالي الصرف (SAR)، عدد المصاريف، النشطون/الكل، نصيب الفرد الوسطي.
- الصرف حسب **التصنيف/المرحلة** | حسب **العملة الأصلية** | **دفع مقابل نصيب** لكل شخص | **منحنى الصرف على الأيام** | ملخّص التسوية | حالة الإقفال (كم أُقفل/متبقٍ).

## 10) التقنية والنشر

- صفحة واحدة `index.html` عربية RTL + **Firebase Firestore** (نفس إعداد مشاريعك) + نشر **GitHub Pages** (`agoulh.github.io/<repo>/`).

## 11) قرارات وتحديات معروفة

- **الدينار العراقي** غير مغطّى في كثير من الـFX APIs المجانية → الإدخال اليدوي هو الافتراضي.
- **العربي في PDF** يحتاج خطًّا مضمّنًا + تشكيلًا صحيحًا → استخدم `html2pdf.js` من HTML جاهز.
- **دقة القسمة:** احسب بالريال واعرض بمنزلتين عشريتين مع تقريب متّسق (تجنّب فروق الهللات في التسوية).
- **استقلال قائمة الرحلة** عن المجموعة المحفوظة إلزامي (snapshot).
- **الإقفال** يجمّد الرصيد ويستبعد من المصاريف اللاحقة، مع إمكانية **إعادة الفتح** عند الحاجة (يلغي الـsnapshot ويعيد الاحتساب).

## 12) خارطة البناء في Claude Code (مراحل)

1. الهيكل + Firebase + قائمة الرحلات + إنشاء رحلة.
2. المجموعات المحفوظة + بناء قائمة المشاركين (كاملة/اختيار/يدوي) + حفظ كمجموعة جديدة.
3. أسعار الصرف (يدوي + جلب لحظي).
4. إضافة/إدارة المصاريف مع تحديد المشاركين (النشطون).
5. محرّك الحساب + الأرصدة.
6. **إقفال الميزانية الجزئي** (معاينة → تأكيد → تجميد → إعادة فتح) + سجلّ `closures`.
7. التسوية (محصّل + أقل تحويلات).
8. الداش بورد (Chart.js).
9. التقارير (Word + PDF) + تصدير الكل.
10. تحسينات + PWA + نشر GitHub Pages.
